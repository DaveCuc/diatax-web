About the Project

A web tool that helps create technical documentation pages automatically. Instead of writing manuals or tutorials by hand, the user enters the link to their project's code (GitHub), and the tool reads the code and writes the guides on its own.

Graph API

Orchestrated Nodes

Built entirely on ADK 2.0. Utilizes distinct function nodes and state edges to achieve predictable, modular workflows instead of simple linear chains.

Sandbox

Secure Isolation

Simulates a secure Google Sandbox by cloning and stripping code repositories of heavy dependencies, environment secrets, and metadata prior to analysis.

Innovation

Writer & Judge Loop

An interactive 3-round critique and refinement loop between specialized Writer and Judge AI agents to guarantee high-quality documentation standards.

The Diátaxis Method

A systematic framework designed around user needs to solve content, style, and structure challenges.

Core Principle

Documentation must be designed around the needs of users throughout their cycle of interaction with a product or technical domain. Created by Daniele Procida.

Methodology Dimensions

Structures content by mapping: Action vs. Cognition (practical doing vs. theoretical knowing) and Acquisition vs. Application (learning a skill vs. applying it on the job).

01

Tutorials

Learning-oriented lessons designed for students to learn by doing under guidance. Omits distractions and theoretical explanations, focusing on repeated visible results.

02

How-To Guides

Goal-oriented recipes providing step-by-step instructions to solve real-world problems. Assumes competence, guiding users safely to complete tasks.

03

Reference

Information-oriented descriptions detailing precise technical facts, APIs, configurations, and specifications neutrally, without opinions.

04

Explanation

Understanding-oriented discussions offering context, background, and connections. Explains the "why" so users can reflect on the architecture.

How it Works

A behind-the-scenes look at the multi-agent generation pipeline

1

Secure Sandbox Initialization

The system clones the public repository into a dedicated folder and recursively removes caches, binary media, and environment secrets.

2

Static Code Investigation

The Researcher agent reads all clean code structures and builds a detailed software architecture mapping, finding entry points and endpoints.

3

Iterative Quality Loop

Writer and Judge agents collaborate in a 3-round review cycle to ensure the documentation conforms strictly to the Diátaxis style guidelines.

4

Site Compilation & Cleanup

The SiteWriter agent translates the markdown document into interactive web page assets, zips them for download, and deletes the workspace.
